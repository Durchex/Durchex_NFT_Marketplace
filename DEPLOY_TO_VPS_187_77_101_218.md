## 🔐 STEP 4: CONFIGURE BACKEND (CRITICAL)

After deployment completes, you MUST edit the backend `.env` file:

```bash
nano /home/durchex/htdocs/durchex.com/backend_temp/.env
```

You'll see a template file. Edit these critical fields:

```env
# 1. MongoDB Connection (REQUIRED)
MONGO_URI=mongodb+srv://YOUR_USER:YOUR_PASSWORD@cluster-name.mongodb.net/durchex?retryWrites=true&w=majority

# 2. JWT Secret (REQUIRED - generate strong random string)
JWT_SECRET=generate_a_strong_random_secret_here_at_least_32_chars

# 3. Smart Contract Addresses (UPDATE with your deployed contracts)
LAZY_MINT_CONTRACT_ADDRESS=0x...
AUCTION_CONTRACT_ADDRESS=0x...
MARKETPLACE_CONTRACT_ADDRESS=0x...

# 4. IPFS Keys (Optional but recommended)
VITE_PINATA_API_KEY=your_pinata_key
VITE_PINATA_SECRET_KEY=your_pinata_secret

# 5. Platform Address
PLATFORM_FEE_RECEIVER=0x... (your wallet address)
```

**To edit**:
- Arrow keys to navigate
- Delete/type to edit values
- `Ctrl + X` → `Y` → `Enter` to save

## 🆘 TROUBLESHOOTING

### Issue: Can't SSH to VPS
```bash
# Test connection
ping 187.77.101.218

# If fails, VPS may be offline or IP wrong
# Contact host provider or verify IP
```

### Issue: 502 Bad Gateway
```bash
# Backend likely not running
pm2 restart durchex-backend

# Check logs for errors
pm2 logs durchex-backend

# Verify on port 3000
curl http://localhost:3000/api/health

# Reload Nginx
sudo systemctl reload nginx
```

### Issue: Frontend not loading
```bash
# Check files exist
ls -la /home/durchex/htdocs/durchex.com/frontend/dist/

# Rebuild if needed
cd /home/durchex/htdocs/durchex.com/frontend
npm run build

# Reload browser (hard refresh: Ctrl+Shift+R)
```

### Issue: SSL certificate not working
```bash
# Check certificate valid
sudo certbot certificates

# Force renewal
sudo certbot renew --force-renewal

# Reload Nginx
sudo systemctl reload nginx
```

### Issue: Database connection fails
```bash
# Check MongoDB credentials in .env
nano /home/durchex/htdocs/durchex.com/backend_temp/.env

# Verify MONGO_URI format is correct
# Test connection from backend logs
pm2 logs durchex-backend
```
